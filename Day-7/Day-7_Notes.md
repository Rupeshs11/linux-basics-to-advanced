# Day 07: Process Management in Linux

## Overview
A process is a running instance of a program in Linux. Process management
commands are used to view, monitor, and control running processes.
These commands help maintain system performance and stability.

---

## Commands

### ps
**Purpose:**  
Used to display currently running processes.

**Syntax:**
- ps
- ps -ef

**Usage:**
- Shows process ID (PID)
- Displays user and command details

---

### top
**Purpose:**  
Used to display real-time running processes.

**Syntax:**
- top

**Usage:**
- Shows CPU and memory usage
- Updates process information dynamically

---

### kill
**Purpose:**  
Used to stop a running process.

**Syntax:**
- kill PID

**Usage:**
- Terminates a process using its PID
- Useful when a process is not responding

---

### jobs
**Purpose:**  
Used to display background jobs.

**Syntax:**
- jobs

**Usage:**
- Shows jobs running in background
- Useful when working with multiple tasks

---

## Why These Commands Matter
- Monitor system performance
- Control unresponsive processes
- Manage system resources efficiently
- Essential for system administrators

---

## Summary
- ps → view running processes  
- top → real-time process monitoring  
- kill → stop a process  
- jobs → view background jobs  

Process management is critical for maintaining a healthy Linux system.
