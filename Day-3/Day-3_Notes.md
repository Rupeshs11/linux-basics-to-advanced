# Day 03: Utility Commands in Linux

## Overview
Utility commands are used to get help about commands, locate executables,
monitor system uptime, and create shortcuts for frequently used commands.
They help improve productivity while working in Linux environments.

---

## Commands

### help
**Purpose:**  
Used to check syntax and available options for shell built-in commands.

**Usage:**
- Shows quick help
- Useful when learning a new command

---

### which
**Purpose:**  
Used to find which executable file is used for a command.

**Syntax:**
- which command_name

**Usage:**
- Helpful when multiple versions of a command exist

---

### uptime
**Purpose:**  
Used to check how long the system or server has been running.

**Usage:**
- Shows system running time
- Displays load average

---

### alias
**Purpose:**  
Used to create shortcuts for long or frequently used commands.

**Example:**
- alias l="ls -ltr"

**Usage:**
- Saves time
- Reduces repetitive typing

---

## Why These Commands Matter
- Improve command-line efficiency
- Help in system monitoring
- Useful in daily Linux work
- Commonly used by system administrators

---

## Summary
- help → command assistance  
- which → executable path  
- uptime → system running time  
- alias → command shortcuts
